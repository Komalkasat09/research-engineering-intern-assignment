# Signal — Narrative Intelligence Platform

> **How did climate activism move from fringe subreddits to mainstream policy
> discourse between 2019 and 2023?**
>
> Signal traces the answer post by post.

## What this is

Signal is an investigative platform for analysing how narratives emerge,
spread, and mutate across social media. It was built as a research assignment
for [SimPPL](https://simppl.org), using a provided Reddit/Twitter dataset
covering climate activism from 2019 to 2023.

The goal is not "what are people saying?" — it is:

> **Who changed the narrative first, how fast did it mutate, and when did
> real-world events amplify it?**

That question requires five analytical lenses working together:

| View | What it answers |
|---|---|
| **Narrative map** | Where do conversations cluster in semantic space? |
| **Timeline** | How fast is each narrative mutating week-over-week? |
| **Spread graph** | Which accounts and subreddits are the distribution hubs? |
| **Stance river** | Is the discourse converging or polarising over time? |
| **Coordinated behavior** | Are any accounts posting suspiciously in sync? |

## The five original contributions

### 1. Narrative velocity index

Most social media dashboards show post volume over time. Signal shows
**narrative velocity** — how fast the *language* used to discuss a topic is
changing each week, measured as the cosine distance between consecutive weekly
topic centroids (mean SBERT embeddings).

High velocity = the narrative is mutating, harder to fact-check.
Low velocity  = the narrative is entrenched, easier to track.

```python
velocity = 1 - cosine_similarity(centroid_week_N, centroid_week_N_minus_1)
```

Key insight: COP26 produced a velocity spike in the "policy debate" cluster
*before* producing a volume spike — the framing changed first, then amplified.

### 2. 5D/2D UMAP split (the right way to cluster)

The BERTopic pipeline clusters on **5D UMAP** (not 2D). UMAP's 2D projection
distorts inter-cluster distances to create visually appealing layouts — running
HDBSCAN on those distorted distances gives wrong cluster assignments.

The 2D pass is a *separate* UMAP transform used only for display.

This is documented in McInnes et al. (UMAP paper, Appendix C) and is one of
the most common mistakes in clustering pipelines for social media data.

### 3. Automated event stitching

The timeline view fetches real-world events from the Wikipedia REST API
(`/feed/onthisday/events`) at runtime, filtered to the dataset date range,
and overlays them as annotation pins on the velocity chart.

This automatically connects the "IPCC AR6 Working Group II" report (Feb 2022)
to the corresponding velocity spike in the "scientific coverage" cluster —
without any manual labelling.

### 4. Coordinated behavior detection

Flagging algorithm: account pairs that share the same URL **and** post within
30 minutes of each other, **more than 3 times** across the dataset.

This is a falsifiable, conservative definition. Known false positives:
moderators cross-posting to sister subreddits, bots reacting to breaking news.
Signal surfaces patterns for human review — it does not attribute intent.

### 5. RAG chatbot with OSINT analyst framing

The "Ask Signal" chatbot uses FAISS semantic retrieval to find relevant posts
*before* asking Claude to answer. The system prompt frames Claude as an OSINT
analyst, instructing it to:
- Always cite specific accounts, timestamps, post volumes
- Distinguish correlation from causation
- Suggest follow-up investigative angles
- Flag innocent explanations for unusual patterns

## Running locally

### Prerequisites

- Node.js 20+
- Python 3.11+
- 8 GB RAM (for the embedding + clustering pipeline)

### Setup

```bash
git clone https://github.com/YOUR_USERNAME/signal
cd signal

# Frontend
npm install

# Python pipeline
pip install -r scripts/requirements.txt
```

### Environment variables

```bash
cp .env.example .env.local
# Add your Anthropic API key to .env.local
```

### Run the pipeline

```bash
# Put the SimPPL dataset at data/raw/dataset.csv (or .json)
bash scripts/run_pipeline.sh --input data/raw/dataset.csv

# For faster testing (GPU + 100k post subsample):
DEVICE=cuda SAMPLE=100000 bash scripts/run_pipeline.sh --input data/raw/dataset.csv
```

### Start the dev server

```bash
npm run dev
# Open http://localhost:3000
```

### Publication Mode (Real Data Only)

Signal defaults to real-data-only behavior in API routes. If required artifacts are missing, endpoints return `503` with a list of missing files instead of silently generating synthetic data.

To explicitly enable demo fallbacks during local development:

```bash
export SIGNAL_ALLOW_SYNTHETIC_DATA=true
```

## Deploying to Vercel

```bash
# Install Vercel CLI
npm i -g vercel

# Set the API key secret
vercel secrets add anthropic-api-key YOUR_KEY_HERE

# Deploy
vercel --prod
```

The precomputed `public/data/*.json` files are committed and served statically.
Only the `/api/chat` endpoint calls the Anthropic API at runtime.

## Tech stack

| Layer | Technology | Why |
|---|---|---|
| Frontend | Next.js 14 App Router | Server components, fast cold starts on Vercel |
| Point rendering | Pixi.js (`@pixi/react`) | WebGL, 800k points at 60fps via ParticleContainer |
| Charts | D3 v7 | Full control over velocity + streamgraph layouts |
| Graph | `react-force-graph-2d` | Force-directed layout, handles 2k+ nodes |
| State | Zustand | Lightweight, no boilerplate, cross-view sync |
| Embeddings | `sentence-transformers/all-MiniLM-L6-v2` | Fast, 384-dim, strong quality/speed trade-off |
| Topic model | BERTopic + HDBSCAN + UMAP | Explainable clusters, c-TF-IDF human-readable labels |
| Stance | `cross-encoder/nli-deberta-v3-small` | Zero-shot, no fine-tuning needed |
| Vector search | FAISS `IndexFlatIP` | Exact cosine search, <100ms at 800k vectors |
| Analytics DB | DuckDB | Fast time-series aggregations, columnar, no server |
| LLM | Claude API (`claude-sonnet-4-20250514`) | Best instruction-following for OSINT framing |
| Deployment | Vercel (frontend) | Free tier, automatic preview deployments |
